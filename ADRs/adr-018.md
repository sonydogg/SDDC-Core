# ADR-018: Standardized Input for Perforce Table Management

**Status:** Accepted (2026-03-16)

**Context:**
The `sddc-p4d` environment lacks a text editor, and the `--replace` flag is unavailable for the `p4 protect` command. To manage server-level configurations (Protections, Depots, Groups), a reliable, non-interactive method is required that works across all Perforce versions.
* **Constraint A:** Avoid reliance on version-specific flags like `--replace`.
* **Constraint B:** Ensure formatting (headers and indentation) matches Perforce requirements.
* **Constraint C:** Use the VS Code terminal for all administrative injections.

**Decision:**
We will utilize the **`p4 [command] -i` (Input)** pattern for all table modifications:
1. **The Format:** Inputs must include the table header (e.g., `Protections:`) followed by indented entries.
2. **The Pipeline:** Configuration will be drafted in VS Code and piped into the container via `docker exec -i`.
3. **Validation:** The `-o` (Output) flag will be used to verify changes without opening an editor.

**Consequences:**
* **Positive:** Universally compatible across different Perforce server versions.
* **Positive:** Eliminates "Editor Not Found" crashes.
* **Negative:** Highly sensitive to whitespace and header formatting; a missing tab can cause a command failure.

---

**Technical Implementation:**
Example for updating the Protections table:
`echo "Protections: \n\tsuper user admin * //..." | p4 protect -i`
