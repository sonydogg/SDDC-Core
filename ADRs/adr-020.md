# ADR-020: Hybrid Version Control Strategy (The "Stones" Workflow)

## Status
**Proposed** (2026-03-21)

---

## Context
We have established a multi-layered storage architecture (The "Stones") on the Intel Mini. This requires a version control strategy that balances **Private Heavy Assets** (ISOs, Binaries, Internal Configs) with **Public Documentation/Code** (ADRs, Docker Compose). 

Previous attempts at a single-repo strategy led to "Permission Wars" and accidental tracking of runtime data.

## Decision
We will adopt a **Hybrid Version Control System (HVCS)** using Perforce (P4D) as the "Librarian" and Git as the "Courier."

1. **Perforce (The Librarian):** The local Source of Truth. It manages the entirety of `/mnt/stones/wind` and `/mnt/stones/water`. It enforces Read-Only protection to prevent accidental edits.
2. **Git (The Courier):** The Public Distribution channel. It manages a filtered subset of the `wind` stone, pushing code and documentation to GitHub.

---

## The Daily Ritual (Operational Manual)

### 1. The Development Cycle (Perforce)
* **Location:** `/mnt/stones/wind`
* **Workflow:** 1. **Check Out:** `p4 edit <filename>` (Removes the Read-Only lock).
    2. **Modify:** Edit the file in VS Code.
    3. **Submit:** `p4 submit -d "Description"` (Re-locks the file and saves to the Earth Stone).
* **Binary Handling:** Use `+S` (Stored) for large files: `p4 add -t binary+S //images/iso/windows/win11.iso`.

### 2. The Publishing Cycle (Git)
* **Location:** `/mnt/stones/wind` (Filtered by `.gitignore`)
* **Workflow:** 1. **Stage:** `git add .`
    2. **Commit:** `git commit -m "Syncing stable updates"`
    3. **Push:** `git push origin main`
* **Constraint:** Never push until the Perforce submit is successful.

## Consequences
* **Integrity:** Files are protected by filesystem locks, preventing "accidental" edits without intent.
* **Storage Efficiency:** Perforce manages large ISOs on the Water stone without bloating the Git history.
* **Privacy:** Secrets in `/mnt/stones/wind/secrets/` and `.env` are explicitly ignored by both `.p4ignore` and `.gitignore`.
