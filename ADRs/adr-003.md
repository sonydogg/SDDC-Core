# ADR-003: Lightweight Hardware Observability via n8n & Postgres

**Status:** Accepted (2026-03-17)

**Context:**
The Intel Mini "Fifth Element" cluster requires hardware health monitoring (CPU Temperature and Storage Utilization across the `/mnt/stones/` tiers). 
* **Constraint A:** Minimal resource overhead on the Intel Mini.
* **Constraint B:** Avoid complex third-party agent installations (e.g., Zabbix, Datadog).
* **Constraint C:** Leverage existing "Earth" (Postgres) and "Wind" (n8n) infrastructure.
* **Constraint D:** Remote accessibility without exposing raw SSH ports to the web.

**Decision:**
We will implement a **Decoupled Monitoring Agent** using n8n workflows:
1. **The Ingestor (Agent):** An n8n cron-scheduled workflow that uses SSH Key Authentication to execute a non-blocking `for` loop on the host. This script extracts disk percentages and thermal millidegrees.
2. **The Ledger (Storage):** A generic Key-Value Time-Series table in PostgreSQL (`sddc_metrics`). This allows for high-frequency writes and easy historical querying without schema changes.
3. **The Thermal Guard (Logic):** An SMTP-based alerting system that bypasses OAuth 2.0 dependencies to provide real-time "High Temp" notifications.
4. **The Portal (Presentation):** An n8n Webhook-to-HTML interface that renders a dark-mode dashboard directly from Postgres data.

**Consequences:**
* **Positive:** No new software needs to be installed on the host OS; if n8n is running, the monitor is running.
* **Positive:** The dashboard is accessible via any browser using the n8n public-facing URL.
* **Negative:** High-frequency polling (e.g., every 1 second) is not feasible due to SSH overhead; 15–30 minute intervals are recommended.
* **Negative:** If the n8n Docker container crashes, the monitoring and alerting both stop (Single Point of Failure).

---

**Technical Implementation Note:**
The data schema follows a "Generic Metric" pattern:
* `category`: (e.g., 'storage', 'processor')
* `metric_name`: (e.g., 'earth', 'wind', 'water', 'CPU Package')
* `metric_value`: (Float)
* `unit`: ('%', 'C')
