# ADR-003: Automated Documentation Pipeline (GitOps)

* **Status:** Approved
* **Date:** 2026-03-15
* **Deciders:** Trevor (Architect)

---

## 1. Context and Problem Statement
Maintaining synchronization between technical decisions in version control and executive-level documentation in Confluence is traditionally a manual, error-prone process ("IT Dinosaur" method).

## 2. Decision Outcome
We have implemented a **GitOps-driven documentation pipeline** using n8n and Cloudflare Zero Trust.

### The Workflow:
1. **Source:** Markdown files authored in VS Code and pushed to GitHub.
2. **Transport:** Outbound Cloudflare Tunnel (Zero Trust) allows GitHub webhooks to reach the internal n8n instance without open ports.
3. **Processing:** n8n fetches the raw file, converts Markdown to HTML, and pushes it to the Confluence API (V2).
4. **Result:** Confluence "SDDC Core" site is updated instantly upon `git push`.

## 3. Consequences
* **Good:** Single source of truth (Git).
* **Good:** Zero manual copy-pasting required.
* **Good:** High security via Zero Trust Tunnel.
* **Bad:** Dependency on n8n service uptime for documentation sync.

Good bye old ways
