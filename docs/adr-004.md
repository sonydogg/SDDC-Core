# ADR 004: Unified Metric Schema for SDDC Observability

**Status:** Accepted (2026-03-16)

**Context:**
The initial monitoring implementation required separate logic for storage and processor data. As the "Fifth Element" (SDDC Core) expands, adding new sensors (RAM, Network, UPS, etc.) would require constant database schema changes (DDL) if using a column-per-metric approach. 
* **Constraint A:** Must support heterogeneous data types (percentages, temperatures, strings).
* **Constraint B:** Must allow for "Schema-less" expansion of new metrics without modifying the Postgres table.
* **Constraint C:** Must remain performant for time-series queries (e.g., "Show me the last 24 hours of Earth tier usage").

**Decision:**
We will implement a **Key-Value Pair (EAV-lite) Schema** for all system metrics. Instead of columns for specific hardware, we use a single row-based ledger:

1. **The Table Structure:**
   * `category`: High-level grouping (e.g., `storage`, `processor`, `memory`).
   * `metric_name`: The specific entity (e.g., `earth`, `wind`, `water`, `CPU Package`).
   * `metric_value`: A `FLOAT` type to store the primary numerical reading.
   * `unit`: The measurement label (e.g., `%`, `C`, `GB`).
   * `recorded_at`: Standard `TIMESTAMP` for time-series alignment.

2. **The Logic Layer (n8n):**
   * The n8n Code Node will normalize all incoming SSH or API data into this standard JSON shape before hitting the Postgres Insert node.

**Consequences:**
* **Positive:** Adding a new "Stone" or a new hardware sensor requires zero database administration; it is handled entirely in the n8n logic.
* **Positive:** Simplifies the Dashboard (Webhook) query using `DISTINCT ON (metric_name)` to get the latest state of the entire cluster in one scan.
* **Negative:** Storing all values as `FLOAT` means non-numerical data (like "Status: OK") requires a separate string column or mapping.
* **Negative:** Aggregating different units in a single SQL query requires careful filtering by the `unit` or `category` column.

---

**Technical Implementation:**
```sql
CREATE TABLE IF NOT EXISTS sddc_metrics (
    id SERIAL PRIMARY KEY,
    category VARCHAR(50),
    metric_name VARCHAR(50),
    metric_value FLOAT,
    unit VARCHAR(10),
    recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
