# ADR-016: Non-Interactive Librarian Provisioning

**Status:** Accepted (2026-03-16)

**Context:**
The `sddc-p4d` Docker image is a minimalist build that lacks common text editors (vi, nano). Standard Perforce administrative commands (e.g., `p4 user`, `p4 protect`) default to an interactive editor mode, which fails in this environment. 
* **Constraint A:** Initialize the server without installing additional packages in the container.
* **Constraint B:** Automate user creation and protection table updates via the CLI.
* **Constraint C:** Ensure the process is repeatable for future "Stone" migrations.

**Decision:**
We will utilize **Standard Input (STDIN) Piping** for all Perforce administrative tasks:
1. **User Creation:** Use the `-i` (input) flag with `p4 user -f` to accept a Hera-doc or piped string.
2. **Protections:** Use the `--replace` flag with `p4 protect` to overwrite the protections table from the command line.
3. **Password Management:** Pipe the host-side secret file directly into `p4 passwd`.

**Consequences:**
* **Positive:** No need to modify the base Docker image or install editors.
* **Positive:** Higher compatibility with n8n automation (which also cannot use interactive editors).
* **Negative:** Requires precise formatting of the input strings (e.g., exact field names like `User:` and `Email:`).

---

**Technical Implementation:**
Command Pattern: `echo "data" | p4 command -i`
