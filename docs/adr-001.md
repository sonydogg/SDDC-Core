# ADR-001: Management Plane Edge Security & SSL Strategy

* **Status:** Approved
* **Date:** 2026-03-15
* **Deciders:** Trevor (Architect)
* **Consulted:** Gemini

---

## 1. Context and Problem Statement

The SDDC requires a centralized management plane (n8n, Perforce, Proxy) accessible via domain names rather than IP addresses. Standard `.local` mDNS records caused conflicts with the UDM Pro and NextDNS CLI. Furthermore, n8n requires secure cookies (HTTPS) for UI stability, necessitating a valid SSL certificate.

## 2. Decision Drivers

* **Security:** Eliminate the need for open firewall ports (80/443).
* **Stability:** Resolve mDNS conflicts between the UDM Pro and the Intel Mini.
* **Professionalism:** Use a registered business domain (`ethicalitllc.com`) for internal services.

## 3. Considered Options

* **Option 1:** HTTP-01 Challenge (Requires Port 80 open to the internet).
* **Option 2:** Self-Signed Certificates (Triggers browser warnings and breaking n8n secure cookies).
* **Option 3 (Selected):** DNS-01 Challenge via Cloudflare API.

## 4. Decision Outcome

Selected **Option 3**. We transitioned the domain **ethicalitllc.com** to **Cloudflare Nameservers** and implemented **Nginx Proxy Manager (NPM)** using the **DNS-01 challenge**.

### Consequences

* **Good:** Production-grade security with a green padlock.
* **Good:** Zero open firewall ports on the UDM Pro.
* **Bad:** Dependency on Cloudflare API availability.
* **Bad:** Requires sensitive API token management in the Docker secrets stack.

---

## 5. Implementation Notes

1. **DNS Management:** Authoritative DNS handled by Cloudflare.
2. **Internal Routing:** NextDNS (UDM Pro) handles local rewrites pointing `*.ethicalitllc.com` to `192.168.0.153`.
3. **SSL Termination:** NPM handles Let's Encrypt wildcard certificates.
