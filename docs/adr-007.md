# ADR-007: Automated Documentation Sync and Recovery Logic

**Status:** Accepted (2026-03-16)

**Context:**
The SDDC Core uses GitHub as the Source of Truth for all Architectural Decision Records (ADRs). These must be mirrored to Confluence for team accessibility and searchability. However, manual deletions or API timeouts on the Confluence side can cause "Orphaned Records" where a document exists in Git but is missing from the presentation layer.
* **Constraint A:** GitHub remains the absolute Source of Truth.
* **Constraint B:** Systems must handle "Side Deletions" (Confluence pages deleted manually) without manual intervention.
* **Constraint C:** Avoid the use of special characters like `&` in titles or bodies to prevent XML/HTML parsing failures in the n8n pipeline.

**Decision:**
We have implemented a **"Check-Then-Act"** recovery pattern within the n8n documentation workflow:

1. **The Discovery Phase:** * A GitHub trigger monitors for new or updated `.md` files in the ADR directory.
   
2. **The Validation Phase (The "If Node"):**
   * Before attempting an update, the workflow queries Confluence for the existence of the page title.
   * An **If Node** evaluates the response from the Confluence Search API.

3. **The Recovery Branch:**
   * **In the event of a match:** The workflow proceeds to the **Update Node** to sync content.
   * **In the event of a "Document Not Found" (Side Deletion):** The workflow "kicks out" to a **Create Node** to re-provision the page from the GitHub source.

**Consequences:**
* **Positive:** High resilience; the documentation system is self-healing and requires zero manual re-syncing.
* **Positive:** Prevents duplicate page errors by verifying existence before creation attempts.
* **Negative:** Increased API overhead (one extra "Search" call per sync event).
* **Negative:** If a page is renamed in GitHub, the recovery logic will create a new page in Confluence, potentially leaving the old-titled page as an orphan.

---

**Technical Implementation:**
The n8n workflow utilizes the `Confluence Node` for searching by title. The recovery logic is triggered whenever the `results` array from the search is empty. Special character scrubbing is handled in the Markdown-to-HTML Code Node.
