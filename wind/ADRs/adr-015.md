# ADR-015: Perforce Librarian Bootstrap and User Initialization

**Status:** Accepted (2026-03-16)

**Context:**
Upon initial deployment or volume recreation of the `sddc-p4d` container, the Perforce service may start in an uninitialized state. In this state, the `admin` user defined in `docker-compose.yaml` does not yet exist in the internal Protections table, leading to "User not found" or "Access denied" errors in VS Code.
* **Constraint A:** Initialize the `admin` user to match the host-side secret file.
* **Constraint B:** Secure the server immediately by moving from an "Open" to a "Protected" state.
* **Constraint C:** Ensure the `P4NAME` (mini-me-p4d) is recognized by the license and configuration.

**Decision:**
We will perform a manual **Superuser Bootstrap** from the Intel Mini CLI:

1. **Create the User:** We will force-create the `admin` user by running a command that bypasses standard login requirements during the first five minutes of server uptime.
2. **Apply Secret:** The password will be set using the `p4 passwd` command, piped directly from `./secrets/p4_admin_pass`.
3. **Set Protections:** A default Protections table will be written to grant the `admin` user `super` access to all stones (`//...`).

**Consequences:**
* **Positive:** Establishes a secure "Root" for the Librarian.
* **Positive:** Unblocks VS Code extension connectivity.
* **Negative:** Requires an interactive session with the Docker container for the initial setup.
* **Negative:** If the `earth` stone data is deleted, this bootstrap must be repeated.

---

**Technical Implementation:**
Bootstrap Command Sequence:
1. `docker exec -it sddc-p4d p4 user -f admin` (Saves the user profile)
2. `cat ./secrets/p4_admin_pass | docker exec -i sddc-p4d p4 passwd admin`
