# ADR 021: SDDC Rebirth - Tiered Storage & Security Lockdown

## Status
Accepted - 2026-03-23

## Context
The previous SDDC-Core deployment suffered from "Friction Locks" caused by mixed State and Logic in the Wind stone. Perforce was attempting to version-lock active SQLite databases (n8n, Nginx Proxy Manager), leading to server instability and "Circular Deadlocks."

## Decision
We performed a "Scorched Earth" reset of the Perforce server to implement a strictly tiered "Stone" architecture.

### 1. Tiered Storage (The Stones)
We unentangled the physical directories on **Intel-01**:
* **Wind (Logic):** Pure text, YAML, and scripts. Tracked by P4 and Git.
* **Earth (State):** Databases and persistent app data. Excluded from version control.
* **Water (Assets):** Large binaries and Perforce Depot history.

### 2. Server Configuration (The Librarian)
* **Image:** `perforce-p4:v25.2`
* **Security Level:** Escalated from `0` to `2`.
* **Authentication:** Passwords now mandatory; sessions managed via Tickets.
* **Initialization:** Used `P4D_SUPER_PASSWD_FILE` to inject the admin password during the first boot (Day Zero).

## Implementation Steps Taken
1.  **Stop Services:** `docker compose down`.
2.  **Migration:** Moved `n8n_data` and `proxy_data` from `/mnt/stones/wind` to `/mnt/stones/earth`.
3.  **Wipe:** Cleared `/mnt/stones/earth/p4data` to remove corrupted metadata.
4.  **Rebuild:** Updated `docker-compose.yaml` with the new Stone paths and Security variables.
5.  **Provision:** Created `wind` and `water` depots via `p4 depot`.
6.  **Mapping:** Created the `sddc_architect_ws` workspace on the M1 to bridge the network.

## Consequences
* **Positive:** Zero friction during `p4 reconcile`. Submissions are instantaneous.
* **Positive:** Logic is now portable and can be pushed to GitHub without exposing dynamic state.
* **Negative:** Requires manual `p4 login` every 12 hours (Security Level 2 requirement).
