# ADR-025: GitHub Branch Protection and Agent Git Workflow

**Status:** Accepted (2026-06-11)

**Context:**

* **Constraint A:** SDDC-Core uses a hybrid VCS model — Perforce is the local source of truth; GitHub (`sonydogg/SDDC-Core`) is the public courier and automation trigger layer.
* **Constraint B:** Two agents (Bert on 02, Ernie on 01) operate against the repository. Only Bert holds write access; Ernie is read-only on GitHub, matching its Perforce posture.
* **Constraint C:** Agent-authored changes require human review before merging to main to prevent unreviewed infrastructure modifications reaching production automation pipelines.

**Decision:**

Enforce branch protection on `main` via GitHub Repository Ruleset "agent-stop" (ID: 17578680) with the following rules:

- No branch deletion
- No force push (non-fast-forward blocked)
- Pull Request required; **1 approving review from `sonydogg` before merge**
- CodeQL code scanning gate: block on high/critical security alerts or errors
- Required status check: self-hosted runner (re-registered under SDDC-56)
- No bypass actors — owner is subject to the same rules

**Git Workflow:**

1. Branch from `main` on `origin` (no fork model): `fix/<issue>-<slug>`, `feat/<issue>-<slug>`, `chore/<slug>`, `docs/<slug>`
2. One issue per PR where possible; PR body includes `Closes #N`
3. Conventional commit subjects: `fix:`, `feat:`, `chore:`, `docs:`
4. Adversarial peer review mandatory for significant infra/config changes (pre-commit)
5. PR follow-up gate: every deferred/TODO/follow-up item in a PR body must link a GitHub issue or carry an explicit `wontfix-rationale:` note before merge
6. Prohibited on shared branches: `--force`, `--no-verify`, `git reset --hard`, amending pushed commits without explicit human approval
7. Only Bert opens branches and PRs. Ernie does not push to GitHub.

**Consequences:**

* **Positive:** All changes to main are human-reviewed; agent autonomy is bounded; audit trail is clean; CodeQL provides continuous security scanning.
* **Negative:** Status check dependency on self-hosted runner means PRs are blocked until runner is operational (SDDC-56 dependency). Ruleset currently has no pinned required reviewer — any approval satisfies the gate until explicitly restricted.

**Technical Implementation:**

- Ruleset enforced at: `https://github.com/sonydogg/SDDC-Core/rules/17578680`
- Required reviewer pinning (`sonydogg`) to be applied under SDDC-56 alongside runner re-registration
- Ernie GitHub credential posture: read-only PAT, no push permissions — enforced at PAT scope, not ruleset
